==================================================================
分支命名规范
==================================================================
命名					简要说明 
master	(主分支，提供给用户使用的正式版本)
dev	(开发分支，永远是功能最新最全的分支)
bug-*	(修复分支，修复线上代码的 bug)
feature-*	(功能分支，某个功能点正在开发阶段)
release-*	(发布版本，发布定期要上线的功能)
